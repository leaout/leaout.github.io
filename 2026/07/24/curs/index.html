Curs：一个个人量化投资平台与涨停打板策略实践

背景

A股市场是一个高度复杂的非线性系统。做量化投资，最难的不是写策略，而是建立一套完整的数据采集、策略执行、风控管理和绩效回测的基础设施。市面上虽然有聚宽、米筐等平台，但本地化部署的可控性、自由度以及实时 Tick 级别的数据处理能力，始终是一个门槛。

这就是 Curs 项目的出发点——一个全栈式个人自动化量化投资平台。项目地址：github.com/leaout/curs

系统架构

Curs 采用单进程架构，Web 服务和交易引擎运行在同一进程中，避免了 IPC 的复杂性和性能损耗：

run.py (单进程) ├── Web 服务 (Flask, 端口 5000) │ ├── 策略管理 (ECharts 可视化) │ ├── 信号查询 │ ├── 股票池管理 │ ├── 持仓监控 │ └── 定时任务配置 └── 交易引擎 (Engine) ├── 策略管理器 (StrategyManager) — 支持热加载 ├── QMT 账户 (QmtStockAccount) — 实盘通道 │ └── 东方财富交易接口 ├── 优化行情引擎 (OptimizedQuoteEngine) └── 事件调度器 (EventsScheduler) ├── 数据采集 (东财热点/同花顺) ├── 定时任务 (Cron/Interval) └── 盘后处理 (盈利统计/数据导入)

关键设计决策

行情优化：用 OptimizedQuoteEngine 只订阅热点股票池（200-500只），而非全市场5000+，Tick 级数据处理延迟控制在3秒以内。

双模式切换：每个策略可在 Web UI 一键切换「实盘模式」和「信号模式」，信号模式只记录买卖信号到数据库不下单，非常适合策略调试和回测验证。

多 Broker 支持：同时支持 QMT 和东方财富交易接口，配置化切换。

涨停打板策略：三层过滤体系

涨停打板的核心是在股票封板瞬间识别有效信号，排除诱多陷阱。Curs 实现了三套逐步迭代的策略，这里重点分析 limit_up_strategy 和 limit_up_strategy_advanced。

第一层：粗筛

策略首先构建了两个核心数据源——昨日涨停股票和热门股票池，并对热点股做了连续上榜过滤，只保留至少连续2天在热点榜上的股票，过滤掉一日游的热点。

`python

从数据库获取昨日涨停股票（排除追高）

yesterday = (datetime.now() - timedelta(days=1)).strftime(‘%Y-%m-%d’)
zt_stocks = db_manager.get_zt_stocks_by_date(yesterday)
context.pre_limit_up_stocks = set(zt_stocks)

从数据库获取热门股票池

hot_stocks = db_manager.get_hot_stocks_from_pool(‘hot’)
context.hot_stocks = set(hot_stocks)

连续上榜过滤：只保留至少连续2天在热点榜上的股票

consecutive = db_manager.get_consecutive_hot_stocks(min_consecutive_days=2)
if consecutive:
context.hot_stocks &= set(consecutive)
`

过滤规则：

  1. 排除 ST 股票
  2. 排除昨日已涨停的股票
  3. 排除不在热门股票池中的股票
  4. 排除当日已触发的烂板（涨停后开板再次封板）
  5. 连续上榜过滤——只保留连续2天热度在榜的股票

第二层：挂单量衰减检测（核心信号）

这是策略最核心的逻辑。当一只股票涨停时，卖一到卖五档口会出现涨停价的卖单。如果主力真的想封板，这些卖单会被快速吃掉。反之，如果挂单量在上涨，说明主力在板上出货。

`python

检查5档行情中是否有涨停价

if upper_limit in ask_prices:
limit_up_index = ask_prices.index(upper_limit)
current_volume = ask_volumes[limit_up_index]
last_volume = context.last_order_volumes.get(stock_code, current_volume)

# 判断挂单量是否减少40%以上
if current_volume <= last_volume * 0.6:
    buy_at_limit_up(context, stock_code, upper_limit)
    context.daily_signaled_stocks.add(stock_code)

context.last_order_volumes[stock_code] = current_volume

`

当挂单衰减率 (last_volume - current_volume) / last_volume >= 40% 时触发买入信号，意味着涨停价卖单被快速消化，封板意图强烈。

策略还有一个排板买入机制——如果某只股票已出现在涨停价的买方队列，但之前已观察到卖单被吃完（last_volume > 0 且当前 current_volume = 0），说明筹码已被扫货，直接追涨买入。

第三层：卖出策略组合

买入只是半场比赛，卖出的节奏决定盈利。策略采用了三层卖出逻辑：

`python
def sell_strategy(context, stock_code, tick):
# 买入后持续追踪峰值价格
max_peak = context.trade_records[stock_code][‘peak_price’]
if current_price > max_peak:
max_peak = current_price
context.trade_records[stock_code][‘peak_price’] = max_peak

# 策略1：动态止盈 — 从峰值回落2%卖出
if current_price < max_peak * 0.98:
    sell_stock(context, stock_code, current_price)

# 策略2：时间窗口 — 10:00-10:30 弱势卖出
# 如果不在涨停状态且走势弱于预期，在10:00-10:30卖出

# 策略3：尾盘强制清仓 — 14:55 必然清仓
if current_time.time() >= datetime.time(14, 55):
    close_position(context, stock_code)

`

这里的关键是峰值追踪，这是日内交易中最重要的变量。策略在买入后持续追踪每只股票的日内最高价，回落2%即触发卖出，这是一种典型的移动止盈策略。

T+1 高级版：第二天智能卖出

A 股是 T+1 制度，涨停打板买入的股票最早要到下一个交易日才能卖出。limit_up_strategy_advanced 对次日卖出做了精细化处理，基于开盘价格区间的决策树来决定是否持有还是卖出：

`python
def analyze_sell_decision(context, stock_code, check_record, tick):
open_price = check_record[‘open_price’]
current_price = check_record[‘current_price’]
change_ratio = (current_price - open_price) / open_price

# 1. 开盘涨停 → 坚决持有
if current_price >= upper_limit:
    return {'action': 'HOLD', 'reason': '开盘涨停，坚决持有'}

# 2. 强势上涨 (>8%) → 继续观察
if change_ratio > 1.08:
    return {'action': 'HOLD', 'reason': '强势上涨，继续观察'}

# 3. 温和上涨 (3%-8%) → 比较买入成本
if 1.03 <= change_ratio <= 1.08:
    if current_price > buy_price:
        return {'action': 'HOLD', 'reason': '价格高于成本'}
    else:
        return {'action': 'SELL', 'reason': '价格回落至成本以下'}

# 4. 弱势/下跌 → 果断卖出
if change_ratio < 1.03:
    return {'action': 'SELL', 'reason': '开盘走势疲弱'}

`

核心假设很简单：涨停股次日开盘走势决定了当天的概率分布。强势开盘（涨停或大涨8%+）的高概率延续是持有信号；温和上涨但价格已经低于成本说明情绪转弱，应果断离场；开盘疲弱直接卖出，不抱幻想。

SVM 机器学习模型

除了规则策略，Curs 还集成了一套 SVM 分类模型进行股价走势预测。模型用过去15个交易日的7个技术因子预测未来5个交易日的涨跌方向：

`python
features = [
close_mean, # 收盘价均值比
volume_mean, # 成交量均值比
max_mean, # 最高价均值比
min_mean, # 最低价均值比
vol, # 当日成交量
return_now, # 15日收益率
std # 收盘价标准差
]

clf = svm.SVC(C=1.0, kernel=’rbf’, gamma=’auto’)
clf.fit(x_train, y_train)
`

每周一用模型预测周五涨跌，预测上涨则开仓、目标10%止盈、回撤2%止损。这个 ML 示例表明 Curs 的架构可以轻松接入任何机器学习模型——只需要实现 handle_tick 接口即可。

数据采集管线

量化交易的数据管线决定了策略质量的上限。Curs 对接了多个数据源：

东方财富热点股票采集：每天盘后自动抓取东方财富的热点板块和个股排行榜，解析 HTML 并清洗入库。同花顺热点作为补充数据源，实现多数据源交叉验证。

QMT 行情订阅：通过迅投 QMT 接口订阅实时 Tick 数据，5档盘口数据是核心策略的输入。优化引擎在非交易时段自动休眠，降低系统开销。

定时任务调度：项目内建 ask_scheduler.py 支持 Cron 表达式和固定间隔两种模式，在 Web UI 灵活配置。Windows 批处理脚本实现完整的开机自启、盘前重启、盘后选股和每日导入流程。

任务 触发时间 说明
CursBoot 登录时 启动 QMT + Curs
CursPreMarket 工作日 08:50 盘前重启 QMT + Curs
CursAfterMarket 工作日 16:00 盘后选股
CursDailyImport 每日 21:00 导入采集数据

Web 监控面板

Curs 的 Web 界面基于 Flask + ECharts 实现，提供完整的策略管理能力：

  • 策略列表显示所有已注册策略及其运行状态，用彩色圆点直观展示启停
  • 指标卡片展示累计收益率、总交易次数、成功率、当日信号数等关键指标
  • ECharts 收益曲线折线图和信号统计饼图，支持30秒自动刷新
  • 交易记录表支持 CSV 导出
  • 一键切换实盘/信号模式，无需修改策略代码

盈利统计与分析

ProfitStats 模块是一个轻量级统计引擎，记录每次交易的完整生命周期，从买入到持续更新价格曲线，再到成交确认和卖出，所有数据持久化到 JSON 文件，并可通过 Web 界面查询。

其中最有价值的分析维度是 ime_to_max_profit——从买入到最大盈利的时间分布。这个指标能告诉你涨停打板后大概多久出现最佳卖点，从而优化止盈参数和卖出的时间窗口设置。
